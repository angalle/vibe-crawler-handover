# 도커 컴포즈 기반 운영 배포 가이드

본 문서는 `backend/`와 `frontend/` 하위의 독립적인 Dockerfile을 통해 이미지를 빌드하고, 이를 `docker-compose.prod.yml`을 이용하여 서버 환경에 배포하는 표준 프로세스를 설명합니다.

특히 백엔드 컨테이너 내부에서 도커 명령어를 실행하기 위한 **DooD (Docker-out-of-Docker)** 설정에 대해 강조합니다.

---

## 1. 아키텍처 개요

1. **이미지 빌드 분리**: `backend/Dockerfile`과 `frontend/Dockerfile`을 각각의 컨텍스트(경로) 기준으로 개별 빌드하여 AWS ECR에 푸시합니다.
2. **배포 오케스트레이션**: 서버에 업로드된 `docker-compose.prod.yml`은 빌드된 ECR 이미지를 단순히 다운로드(Pull) 받아 컨테이너들을 엮어서 실행하는 역할을 수행합니다.

---

## 2. 도커 제어 핵심 설정 (DooD) 🚨

백엔드 컨테이너는 동적으로 새로운 크롤러 도커 컨테이너를 실행하고 제어해야 합니다. 이를 위해 다음 두 가지 필수 조건이 설정되어 있습니다.

1. **Docker CLI 설치 (`backend/Dockerfile`)**: 백엔드 내부에서 도커 데몬을 실행하지 않고 껍데기 명령어(`docker-ce-cli`)만 가볍게 설치합니다.
2. **도커 소켓 마운트 (`docker-compose.prod.yml`)**: 호스트 서버(EC2)의 도커 데몬 소켓(`/var/run/docker.sock`)을 백엔드 컨테이너 내부로 볼륨 마운트합니다. 컨테이너 내부에서 실행하는 도커 명령어는 이 소켓을 타고 **실제 호스트 서버의 도커 데몬**을 조종하게 됩니다.

---

## 3. 운영 배포 설정 파일 (`docker-compose.prod.yml`)

운영 서버에 올라가는 핵심 배포 설정 파일입니다. 백엔드에서 DooD 환경을 위한 볼륨 바인딩(`- /var/run/docker.sock:/var/run/docker.sock`)이 구성되어 있음을 확인할 수 있습니다.

```yaml
version: "3.8"

# =================================================================
# 운영(Production) 환경 전용 Docker Compose 파일
# - 로컬 DB 미포함 (외부 DB 사용)
# - AWS ECR 이미지 사용
# - 환경변수는 배포 시 .env 또는 Shell Env에서 주입받음
# =================================================================

services:
  # Migration 서비스 (필요 시 주석 해제)
  # migration:
  #   image: ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-2.amazonaws.com/vibe-migration:latest-prod
  #   container_name: vibe-crawler-migration
  #   command: alembic -c /app/alembic.ini upgrade head
  #   environment:
  #     DATABASE_URL: ${DATABASE_URL}
  #   networks:
  #     - vibe-network
  #   restart: "no"

  # Backend API 서버
  backend:
    image: ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-2.amazonaws.com/vibe-backend:latest-prod
    container_name: vibe-crawler-backend
    ports:
      - "8080:8080"
    environment:
      DATABASE_URL: ${DATABASE_URL}
      PROJECT_ROOT: /app
      SYSTEM_STORAGE_ROOT: /app/system_storage
      HOST_SYSTEM_STORAGE_ROOT: ${HOST_SYSTEM_STORAGE_ROOT:-/home/ubuntu/vibe-deployment/system_storage}
      CORS_ORIGINS: ${CORS_ORIGINS:-*}
      OPENAI_API_KEY: ${OPENAI_API_KEY}
      PYTHONUNBUFFERED: 1
      ENV: prod
      TZ: Asia/Seoul
      MAIL_API_URL: ${MAIL_API_URL:-http://api.solu-tion.co.kr:8080/query}
      SMTP_SENDER_EMAIL: ${SMTP_SENDER_EMAIL:-hslee@mindknock.com}
    volumes:
      - ${HOST_STORAGE_PATH:-/home/ubuntu/vibe-deployment/system_storage}:/app/system_storage
      - backend_logs:/app/logs
      
      - /var/run/docker.sock:/var/run/docker.sock
    # depends_on:
    #   migration:
    #     condition: service_completed_successfully
    networks:
      - vibe-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    restart: unless-stopped

  # Frontend (Vue.js + Nginx)
  frontend:
    image: ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-2.amazonaws.com/vibe-frontend:latest-prod
    container_name: vibe-crawler-frontend
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      backend:
        condition: service_healthy
    networks:
      - vibe-network
    restart: unless-stopped

networks:
  vibe-network:
    driver: bridge

volumes:
  backend_logs:
    driver: local
```

이 파일을 기반으로 서버 배포 스크립트는 `docker compose pull` 및 `docker compose up -d`를 수행하여 컨테이너 아키텍처를 가동합니다.
